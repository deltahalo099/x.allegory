Encoding--------------------------

Any type of experience can be encoded into bits
by some transducer. We take those bits and store
them in little memory containers and give
them structure.

A curriculum is simply the sequential acquistion
of preserved accumulated sense data
(text books, microscope slides, films, memoirs, lab results, etc.)
Any "field of study" or line of inquiry is a large collection of data
associated with some particular phenomena or aspect of reality.
every field has its current state, as well as a history, and they are continually
changing, and crosspolinating and dying out.

In an infinite library one may find things that
are both meaningful and not. It is not the data that has
inherent meaning, it is the experiencer of the information
that gives it relvance and frames it in some context.

information can simply be thought of as some form - some momentary
configuration of the present or past or future state of reality
(be it subjective or objective)

Whether this reality fundamentally is granular; like sand
or continuous; like water, is still an open question because
no one has come close to exploring the planck scale.

A sufficiently intelligent librarian could scour the library
and retrieve data stored and shared across all machines,
(think accessing digitized archives from around the globe)

The library itself can be coded as a simple markup that restructures
any given data into an aesthetically pleasing and intuitively interactive
form.

every "book" can be given a set of metainformation tags to help in catalogging
(author, title, date, language, location, topics, content, themes, genre, etc)
and associating content with branches of knowledge. The content of individual pages
can be indexed and tagged as well to aid in the search for relevent phrases and questions.

additionally - given the immense processing power available and on such small scales
one could easily imagine creating motionpicture like 3d scans of objects, people, professors
to be preserved and archived.

- picture a 3d bust of richard feynman giving his famous lectures on physics instead of
a youtube video.

The library itself is organized in a fractal-like, tree like heirarchy
codex>branches>floors>books>page>(line, word, glyph, bits)

seeing as the method of search is based on metadata and indexing, the need
for a concrete classification system is unnecessary, so long as the illusion of order
and shelving is given - then when the specific data is requested it can simply "appear"
- this is a point i'm not entirely sure on however

basic datastructure types i imagine would include;
.shelf = similar to a folder - but closer to a briefcase or a drive perhaps
.book = equivalent to what we now call folders, large compilations of relevant project data
.page = these would be the main units, marked up almost like html and css, you could have
embedded video, web articles, icons, sketches, hand writing data, sound files, 3d "pop out" objects etc.
